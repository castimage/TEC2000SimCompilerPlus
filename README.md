# TEC2000SimCompiler

TEC2000SimCompiler 是一个用于 TEC-2000Sim 汇编平台的汇编编译器，可以将汇编语言源代码编译成可在 TEC-2000 模拟器上运行的二进制文件从而脱离TEC2000Sim平台限制，以方便开发。

## 功能特点

- 将 TEC-2000 汇编语言编译为二进制可执行文件（.cod 格式）
- 支持语法检查和错误报告
- 提供彩色日志输出便于调试
- 支持普通标签、位置跳转标签与内存直接操作
- 可配置的内存区域管理

## 安装依赖

在运行此编译器之前，您需要安装以下 Python 包，其将用于支持彩色输出：

```bash
pip install colorama
```

## 使用方法

### 基本用法

```bash
python TEC2000SimCompiler.py -i <输入文件> [-o <输出文件>] [-s <起始地址>] [其他选项]
```

> 或使用编译好的exe文件使用相同参数编译相关文件

### 命令行参数

|           参数           | 简写 |                      描述                      |
|:----------------------:|:--:|:--------------------------------------------:|
|      --input-file      | -i |                必需，要编译的汇编源文件路径                |
|     --output-file      | -o |         可选，输出文件路径，默认为输入文件同名的 .cod 文件         |
|    --start_location    | -s |            可选，程序初始写入地址，默认为 0x2000            |
|       --verbose        | -v |                   显示详细编译信息                   |
|        --quite         | -q |                 静默编译，仅在出错时输出                 |
| --allow_non_user_area  | 无  | 允许写入非用户区（0x0000-0x1FFF, 0x2600-0xFFFF），默认不允许 |

### 示例

```bash
# 基本编译
python TEC2000SimCompiler.py -i program.code

# 指定输出文件和起始地址
python TEC2000SimCompiler.py -i program.code -o output.cod -s 0x2100

# 详细输出模式
python TEC2000SimCompiler.py -i program.code -v

# 静默编译
python TEC2000SimCompiler.py -i program.code -q
```

## 汇编语言规范

### 标签样式
- 普通标签：`tag-[0-9a-zA-Z_]+`
- 位置跳转标签：`[0-9a-fA-F]{1,4}:`

### 参数样式
- register(寄存器)：`R0-R15`（受限于实际软件改动，可用 R0-R3, R6-R15）
- memory(内存)：`[0000-FFFF]`
- io_port(I/O 端口)：`00-99`
- address(地址)：`0000-FFFF`
- immediate(立即数)：`0000-FFFF`
- tag_label(普通标签)：`tag-[0-9a-zA-Z_]+`
- offset_jump(跳转偏移)：`+[0-7][0-9a-fA-F]|+80|-[0-7][0-9a-fA-F]`  # **自设**

### 支持的指令

#### 数据传送指令
- `mvrd R, DATA`* - 将16位立即数传送到寄存器
- `mvrr R1, R2` - 寄存器间数据传送
- `ldrr R1, [R2]` - 从内存加载数据到寄存器
- `strr [R1], R2` - 存储寄存器数据到内存

#### 算术运算指令
- `add R1, R2` - R1 = R1 + R2
- `sub R1, R2` - R1 = R1 - R2
- `inc R` - 寄存器值加1
- `dec R` - 寄存器值减1

#### 逻辑运算指令
- `and R1, R2` - 按位与运算
- `or R1, R2` - 按位或运算
- `xor R1, R2` - 按位异或运算
- `shl R` - 逻辑左移一位
- `shr R` - 逻辑右移一位
- `test R1, R2` - 测试寄存器内容
- `cmp R1, R2` - 比较两个寄存器

#### 控制转移指令
- `jr ADDR`* - 相对跳转
- `jrc ADDR`* - 进位时跳转
- `jrnc ADDR`* - 无进位时跳转
- `jrz ADDR`* - 零标志置位时跳转
- `jrnz ADDR`* - 零标志清零时跳转
- `jmpa ADDR`* - 绝对跳转
- `cala ADDR`* - 调用子程序
- `ret` - 返回

#### 堆栈操作指令
- `push R` - 将寄存器压入堆栈
- `pop R` - 从堆栈弹出到寄存器
- `pshf` - 将标志位压入堆栈
- `popf` - 从堆栈弹出标志位

#### I/O 操作指令
- `in PORT` - 从指定端口读取数据
- `out PORT` - 向指定端口输出数据

#### 监控程序提供的子程序
- `inch` - 输入数据
- `out1ch` - 输出字符
- `upcase` - 将字符转为大写
- `indat` - 从键盘输入数据
- `wstr1ch` - 输出字符串
- `inline` - 输入一行数据
- `shdw` - r0 > 8bit
- `shd4` - r0 > 4bit
- `shup` - r0 < 8bit
- `shu4` - r0 < 4bit
- `numasc` - 输出R15的十六进制数字

#### 自设指令
- `dw DATA` - 定义16位数据

> 注：  
> 1. 标 * 号的指令可用tag标签作为参数  
> 2. jump系指令可使用offset_jump参数


### 寻址方式

1. **寄存器寻址**：`R0-R15`（受限于实际硬件，可用 R0-R3, R6-R15）
2. **立即数寻址**：4位十六进制数
3. **直接寻址**：4位十六进制地址
4. **寄存器间接寻址**：`[R0-R15]`
5. **I/O端口寻址**：0-99 的十进制端口号

### 标签和注释

- 跳转标签格式：`地址:` （例如 `2000:`）
- 注释以 `#` 开头，可定义在语句末尾或空行中

### 示例程序

```asm
# 简单程序示例：将数值1234存入R1寄存器
2000:           # 程序入口地址
mvrd R1, 1234   # 将1234载入R1寄存器
ret             # 返回
```

## 技术细节

### 文件格式

生成的 .cod 文件包含以下结构：
1. **文件头**（5字节，小端序）：
   - 标志位（1字节）固定为0x01
   - 起始地址（2字节）
   - 长度（2字节）
2. **程序体**（n字节，大端序）：
   - 编译后的机器码

### 内存管理

- 默认用户区地址范围：0x2000 - 0x25FF
- 最大地址空间：0x0000 - 0xFFFF
- 支持分段编程，可以定义多个代码块

### 编译过程

编译过程分为以下几个步骤：
1. **语法检查**：验证汇编源文件的语法正确性
2. **预处理**：解析跳转标签，组织代码块
3. **地址分配**：确定各代码块在内存中的位置
4. **代码生成**：将汇编指令转换为机器码
5. **文件输出**：生成带文件头的二进制文件

### 特殊处理

1. **相对跳转偏移计算**：
   - 对于 `jump` 系列的单字指令，编译器会自动计算相对于当前指令的偏移量
   - 偏移量计算公式：目标地址 - (当前地址 + 1)
   - 如果偏移量超出 -128 到 127 的范围，会抛出跳转错误

2. **寄存器编码**：
   - 不同指令中寄存器的编码方式不同
   - `p0` 格式：寄存器号放在高4位（如 dec, inc, shl, shr, pop, mvrd）
   - `0p` 格式：寄存器号放在低4位（如 push）

3. **普通标签处理**：
   - 预处理时将普通标签存储到全局字典，键为标签名，值为标签实际内存地址
   - 编译时将标签名替换为标签实际内存地址，确保指令正确无误
   - 如果标签未定义或定义位置错误，会抛出标签错误异常

## 错误处理

编译器提供多种错误检测机制：
1. **语法错误检测**：检查指令拼写、参数数量和格式
2. **地址越界检查**：确保所有地址在有效范围内
3. **跳转范围验证**：检查相对跳转指令的目标地址是否可达
4. **参数格式验证**：验证各种寻址模式的参数格式

## 开发信息

该编译器使用 Python 3 编写(开发版本3.12)，主要利用了以下模块：
- `argparse`：命令行参数解析
- `colorama`：跨平台彩色终端输出
- `struct`：二进制数据打包
- `re`：正则表达式处理

### 架构设计

编译器采用面向对象设计，主要包括以下类：

1. **T2kSCompiler**：主编译器类，负责整个编译流程
2. **MemRangeManager**：内存范围管理器，处理内存区域分配
3. **CLogger**：日志类，提供彩色日志输出
4. **异常类**：一系列自定义异常处理不同类型的错误

### 日志系统

采用四等级日志系统：
- DEBUG：详细调试信息
- INFO：一般信息
- WARNING：警告信息
- ERROR：错误信息

**程序使用了装饰器为函数添加调试信息日志，日志等级默认为DEBUG，可通过携带'-v'或'--verbose'编译参数查看DEBUG日志**

## 注意事项

1. 默认只允许在用户区（0x2000-0x25FF）编写程序
2. 跳转指令分为相对跳转和绝对跳转，相对跳转有距离限制
3. 某些寄存器可能在实际硬件中有特殊用途，请参考 TEC-2000 文档
4. 程序必须有明确的结束方式（如ret指令），否则会继续执行后续内存中的数据
